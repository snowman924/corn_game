# 말랑 옥수수의 핫팬 탈출! (Soft Corn's Hot Pan Escape)
### 테스트 계획서 (Verification & Test Plan) - *자동화 테스트 보강 버전*

본 테스트 계획서는 게임 개발 과정 및 완료 후 품질을 보증하기 위해 **자동화 테스트(CLI)**와 **수동 테스트(Manual UI/UX)**의 영역을 명확히 구분하여 테스트 시나리오를 구성합니다.

---

## 1. 테스트 방식의 이원화 (Automated vs. Manual)

```
                              [ 테스트 방식 결정 ]
                                        |
                 +----------------------+----------------------+
                 |                                             |
                 v                                             v
        [ 🤖 자동화 테스트 ]                           [ 🖐️ 수동 테스트 ]
  - Node.js 환경에서 CLI로 즉시 실행             - 모바일 실기기 및 크롬 DevTools 사용
  - DOM/브라우저가 필요 없는 핵심 로직           - 그래픽 연출, 사운드, 모바일 조작 감도
  
  * 대상:                                      * 대상:
    1. 플레이어 이동 경계선 계산                 1. 인트로 웹툰 렌더링 & 슬라이드 전환
    2. 엔티티 간 원형/AABB 충돌 공식             2. 피격 시 화면 붉은 흔들림(Shake) 효과
    3. 적 스폰 주기 및 발사 각도 계산             3. HP 0 도달 시 팝콘 폭발 파티클 연출
    4. 스킬 시전 효과 및 버프 적용               4. 가상 조작패드(Joystick) 터치 반응성
    5. 상점 영구 업그레이드 & 코인 계산          5. 브라우저 탭 비활성화 시 자동 일시정지
```

## 2. 자동화 테스트 계획 (Automated Tests Specification)

모든 자동화 테스트는 브라우저 없이 Node.js CLI를 사용하여 로컬 환경에서 실행할 수 있습니다. 

### 2.1. 실행 방법
```bash
# 전체 단위 테스트 실행 명령어 (Node.js 가 설치되어 있어야 함)
node test/runTests.js
```

### 2.2. 자동화 테스트 세부 시나리오

#### `gameEngine.test.js` (게임 핵심 루프 및 물리)
*   **T-AUTO-01: 경계선 탈출 방지**
    *   *설명*: 플레이어가 프라이팬 가상 영역인 가로 `800px`, 세로 `450px` 범위를 벗어날 수 없는지 확인합니다.
    *   *검증*: 플레이어의 좌표가 최댓값에 수렴할 때 외곽 경계선에 부딪혀 좌표 증가가 멈추는지 검증합니다.
*   **T-AUTO-02: 충돌 판정 알고리즘**
    *   *설명*: 2D 원형 충돌(Circle-Circle Collision) 알고리즘이 올바르게 닿았을 때 `true`를 반환하는지 확인합니다.
    *   *검증*: 두 점의 좌표 차이가 반경 합보다 작거나 같을 때 충돌 감지가 작동하는지 검증합니다.
*   **T-AUTO-03: 시간 흐름에 따른 승리 조건**
    *   *설명*: 플레이어 체력이 유지된 채로 60초가 흐르면 게임 상태가 `PLAYING`에서 `VICTORY`로 변경되는지 검증합니다.
    *   *검증*: 엔진의 `update(60)`를 호출한 뒤 `engine.status`가 `'VICTORY'`인지 검증합니다.

#### `entities.test.js` (캐릭터, 적, 스킬 논리)
*   **T-AUTO-04: 열 상태 단계별 캐릭터 상태 변경**
    *   *설명*: 플레이어 HP 비율에 따라 `heatState` 변수가 정확히 바뀌는지 검증합니다.
    *   *검증*: HP 100(단계 1) -> HP 60(단계 2) -> HP 30(단계 3) -> HP 0(단계 4, 사망)에 알맞은 코드값이 리턴되는지 검증합니다.
*   **T-AUTO-05: 소금 뿌리기 스킬 범위 내 적 제거**
    *   *설명*: 소금 뿌리기 시전 시 범위(120px) 바깥의 적은 생존하고, 안쪽의 적만 제거되는지 검증합니다.
    *   *검증*: 범위 100px 거리에 있는 기름 병정은 사망 상태(`isDead = true`)로 변경되고, 150px 거리의 기름 병정은 정상 생존 상태를 유지하는지 확인합니다.
*   **T-AUTO-06: 아이스 팩 방어 버프**
    *   *설명*: 아이스 팩 사용 후 2초 동안 가해지는 데미지가 50% 반감되는지 검증합니다.
    *   *검증*: `player.useSkill('ice')` 호출 후 데미지 입력값 20이 주어질 때 실제 체력 차감이 10만 발생하는지 확인합니다.
*   **T-AUTO-07: 대시 스킬 무적 판정**
    *   *설명*: 대시 스킬 활성화 상태 동안은 피격 데미지가 전혀 들어가지 않는지 검증합니다.
    *   *검증*: 대시 상태 변수가 활성화되었을 때 충돌 함수가 동작해도 플레이어 HP가 깎이지 않음을 확인합니다.

#### `shopManager.test.js` (데이터 세이브 및 상점 구매)
*   **T-AUTO-08: 코인 잔액 부족 시 구매 차단**
    *   *설명*: 사용자의 보유 코인보다 높은 가격의 업그레이드를 시도하면 구매를 거절하고 에러가 발생하는지 검증합니다.
    *   *검증*: 코인 10개 보유 상태에서 100코인짜리 HP 강화를 요청할 때 에러가 리턴되고 코인이 보존되는지 확인합니다.
*   **T-AUTO-09: 세이브 데이터 데이터 정합성**
    *   *설명*: 데이터 세이브 및 로드 시 정해진 스키마에서 어긋나지 않는지 검증합니다.
    *   *검증*: 저장된 데이터 필드에 `coins`, `upgrades`, `unlockedSkills` 등의 필수 키 값이 누락 없이 보존되는지 검증합니다.

## 3. 수동 테스트 계획 (Manual Tests Specification)

### 3.1. 모바일 UI/UX 기기 테스트
*   **T-MAN-01: 반응형 웹 뷰포트**: Chrome DevTools 모바일 에뮬레이터를 활용해 가로/세로 화면 전환 시 글래스모피즘 모달 박스가 가운데에 정렬되며 버튼 겹침 현상이 발생하지 않는지 검증합니다.
*   **T-MAN-02: 가상 조작패드 피드백**: 터치한 손가락의 드래그 반경만큼 조작패드의 안쪽 스틱이 잘 따라오는지, 터치를 떼면 중립 상태로 신속하게 복귀하는지 직접 확인합니다.

### 3.2. 애니메이션 및 그래픽 연출
*   **T-MAN-03: 피격 시 화면 붉은 흔들림**: 데미지를 입었을 때 Canvas의 `ctx.translate`를 사용해 화면이 0.2초간 격렬히 흔들리고, 가장자리에 붉은 그림자(Vignette) 효과가 깜빡이는지 눈으로 확인합니다.
*   **T-MAN-04: 팝콘 변신 효과**: 캐릭터 HP가 0이 되는 순간 사방으로 노란 불꽃과 함께 하얗고 귀여운 팝콘 이미지가 팡! 터지며 상승하는지 검증합니다.

### 3.3. 브라우저 예외 상황
*   **T-MAN-05: 비활성화 일시정지**: 인게임 상태에서 브라우저 창을 최소화하거나 다른 탭을 열었다가 다시 돌아왔을 때, 게임 타이머 및 적들이 중단된 순간부터 다시 재개되는지 수동으로 검증합니다.
