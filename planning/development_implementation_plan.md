## 2. 자동화 테스트 구현 세부 가이드 (Test Implementation Steps)

### 1단계: M2 단계의 테스트 러너 환경 설정 (`test/runTests.js`)
*   별도의 중량급 프레임워크(Jest 등) 의존성을 최소화하고 웹 프런트엔드에서 즉시 동작 가능한 **경량 커스텀 테스트 러너**를 구현합니다.
*   이 러너는 Node.js 환경에서 `node test/runTests.js` 명령어로도 수행될 수 있고, 브라우저 콘솔에서도 동작할 수 있도록 범용 모듈 스타일로 작성합니다.
*   각 테스트 블록(`describe`, `it`, `assert`) 함수를 작성하여 컬러풀하게 터미널에 테스트 성공/실패 여부를 출력합니다.

### 2단계: 게임 물리 엔진 테스트 (`test/gameEngine.test.js`)
*   **이동 경계선 검증**:
    *   플레이어의 위치가 `x = 800`, `vx = 10` 일 때 한 프레임 업데이트 후 플레이어의 `x`가 `800` 경계선을 넘지 않고 멈춰 있는지 코드로 확인.
*   **충돌 검증**:
    *   플레이어(반경 15px)와 기름 방울(반경 5px)의 거리가 18px일 때 충돌로 판정되어 플레이어 HP가 감소하는지 검증.

### 3단계: 상점 데이터 및 세이브 테스트 (`test/shopManager.test.js`)
*   코인이 100개 있을 때 150코인짜리 대시 스킬 해금을 시도하면 에러를 반환하고 세이브 데이터가 변하지 않는지 검증.
*   정상 구매 후 코인이 정확히 차감되어 가상 스토리지에 세이브되는지 확인.
