# GEMINI Developer Guide: E2E Visual 검증 규정

본 프로젝트는 모바일 반응형 뷰포트(가로/세로 가변 해상도)를 전면 지원하는 인터랙티브 웹 게임입니다. 로직 회귀 검증뿐만 아니라, **화면 레이아웃 및 픽셀 종횡비 깨짐을 방지하는 E2E 비주얼 검증(Visual Regression Test)**이 필수적으로 요구됩니다.

다음 작업자(AI 에이전트)는 UI/CSS/HTML 관련 코드를 수정할 때 반드시 본 규정에 따라 검증을 수행하고 리포트해야 합니다.

---

## 1. E2E 비주얼 검증 환경 개요

프로젝트에는 Puppeteer 헤드리스 브라우저를 기반으로 로컬 웹서버를 자동으로 구동하여 기기별 시나리오를 캡처하는 검증 도구가 구축되어 있습니다.

- **실행 대상 스크립트**: [test/capture.js](file:///home/sangtai/git-repo/corn_game/test/capture.js)
- **주요 시나리오**: `index.html` 로드 (인트로 화면) ➔ 인트로 스킵 버튼 클릭 (로비 화면) ➔ 게임 시작 버튼 클릭 (인게임 플레이 화면)
- **시뮬레이션 해상도**:
  - **세로 모드 (Portrait)**: 390x844 (iPhone 14 Pro 비율)
  - **가로 모드 (Landscape)**: 844x390 (iPhone 14 Pro 가로 비율)

---

## 2. 검증 실행 명령어

UI 수정을 완료한 후, 터미널에서 다음 명령어를 실행하여 최신 렌더링 스크린샷들을 갱신하십시오.

```bash
node test/capture.js
```

실행이 완료되면 루트 디렉토리에 아래의 스크린샷 이미지 파일들이 자동 생성 및 덮어쓰기됩니다.
- 세로 모드 E2E 단계별 캡처:
  - `screenshot_portrait_1_intro.png`
  - `screenshot_portrait_2_lobby.png`
  - `screenshot_portrait_3_game.png`
- 가로 모드 E2E 단계별 캡처:
  - `screenshot_landscape_1_intro.png`
  - `screenshot_landscape_2_lobby.png`
  - `screenshot_landscape_3_game.png`

## 3. 에이전트의 Vision 판독 및 검증 포인트

생성된 이미지 파일들을 `view_file` 도구를 사용하여 직접 열고, 시각적으로 판독하여 아래 항목들을 검증해야 합니다.

1. **캔버스 해상도 비율 보존**: 
   - 캔버스 안의 정원 형태(가열 코일 링 등)가 타원형으로 찌그러지지 않고 완벽한 원형을 유지하는가?
   - 플레이어 캐릭터(옥수수 🌽) 및 적군의 그래픽 픽셀이 1:1 종횡비를 지키는가?
2. **세로모드 전체 화면화 (100vh) 및 오버레이**:
   - 캔버스 배경이 세로 뷰포트 바닥끝까지 완전히 가득 메우며 팽창했는가?
   - 가상 조이스틱과 스킬 버튼들이 캔버스 화면 밑단 좌우 구석에 absolute 형태로 정상 포개어져 렌더링되었는가?
3. **HUD 및 버튼 배치 상태**:
   - 일시정지 아이콘(⏸)이 세로/가로 모드에서 각각 타이머나 HP 바 등 HUD 구성요소와 겹치지 않고 자연스럽게 놓여있는가?

## 4. 리포트 가이드라인

작업 완료 후 작성하는 `walkthrough.md`에 E2E 검증 결과 이미지 링크를 명시하고, 비주얼 판독 결과를 상세하게 리포트하십시오.

> [!NOTE]
> E2E 비주얼 검증을 생략하거나 검증 결과 이미지를 직접 판독하지 않고 작업을 종료하는 것은 허용되지 않습니다.
